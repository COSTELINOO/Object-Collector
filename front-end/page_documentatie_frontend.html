1. Introducere

Această documentație oferă o prezentare completă a implementării front-end a aplicației Collect.me. Aplicația este un sistem de gestionare a colecțiilor care permite utilizatorilor să creeze, să administreze și să partajeze colecțiile lor compuse din diverse tipologii de obiecte. Documentul detaliază structura, modelele de design, tehnologiile utilizate și funcționalitățile componentelor front-end.

2. Prezentare generală a aplicației

Collect.me este o aplicație web concepută pentru a ajuta colecționarii să organizeze și să-și prezinte colecțiile. Funcționalitățile principale includ:

  • Autentificarea utilizatorilor (login/înregistrare cu nume de utilizator sau email)
  • Gestionarea colecțiilor (creare, vizualizare, editare, ștergere)
  • Gestionarea obiectelor (adăugare, vizualizare, editare, ștergere în cadrul colecțiilor)
  • Explorare (vizualizare și căutare în colecții și obiecte publice)
  • Statistici (generare și vizualizare statistici despre colecții și obiecte)
  • Gestionarea profilului utilizatorului
  • Funcționalitate de export (export date colecții în PDF, CSV)

Aplicația este proiectată cu o interfață curată, prietenoasă pentru utilizator, care face gestionarea colecțiilor intuitivă și plăcută.

3. Arhitectură tehnică

Tehnologii utilizate pentru Front-End:

  • HTML5: Oferă fundația structurală pentru toate paginile
  • CSS: Gestionează stilizarea și designul responsive
  • JavaScript: Administrează interactivitatea, validarea formularelor și comunicarea API
  • Fetch API: Utilizat pentru efectuarea cererilor HTTP către back-end

6. Sistem de autentificare

Front-end-ul implementează un sistem de autentificare care interacționează cu endpoint-urile /auth ale back-end-ului.

  • Login: trimite credentiale la /auth/login, primește token JWT, stocat în localStorage
  • Înregistrare: validare client-side, trimitere date la /auth/register
  • Middleware: auth-middleware.js verifică token-ul și redirecționează dacă lipsește
  • Resetare parolă: flux cu email și cod de resetare
  • Logout: eliminare token, notificare backend

8. Integrarea cu Back-End

Front-end-ul comunică cu back-end-ul prin endpoint-uri API RESTful.

Endpoint-uri pentru autentificare

  • POST /auth/login, POST /auth/register, POST /auth/reset-password, POST /auth/logout etc.

Endpoint-uri pentru colecții și obiecte

  • GET /user-collection, POST /user-collection, PUT /user-collection/{id}, DELETE /user-collection/{id}
  • GET /objects, POST /my-collection/{id}, PUT /my-objects/{id}, DELETE /my-objects/{id}

Endpoint-uri pentru statistici

  • GET /statistics/personal/pdf, GET /statistics/public/csv, GET /statistics/public/rss

Fluxul cererilor API

  1. Pregătirea cererii: adăugare header Authorization cu token JWT
  2. Executarea cererii: trimitere către server
  3. Procesarea răspunsului: parsare JSON, verificare coduri de eroare
  4. Gestionarea erorilor: afișare mesaje prietenoase, logica de reîncercare

9. Gestionarea erorilor

Validare client-side

  • Validare câmpuri obligatorii, format email, complexitate parolă
  • Sanitizarea input-ului pentru prevenirea XSS

Gestionarea erorilor API

  • Tratare status HTTP: 400, 401, 403, 404, 500
  • Detectarea problemelor de conectivitate
  • Feedback pentru utilizator: mesaje de eroare și succes clare

10. Design Responsive

  • Breakpoint-uri: Mobile (<767px), Tabletă (768-999px), Desktop (>1000px), Desktop mare (>1200px)
  • Grile fluide, imagini flexibile, media queries pentru layout adaptiv
  • Ascunderea/afișarea elementelor după necesități

11. Caracteristici de accesibilitate

  • Etichetare corectă a formularelor
  • Contrast ridicat pentru text și elemente UI
  • Suport pentru navigare cu tastatura

12. Considerații de securitate

Securitate client-side

  • Prevenirea XSS: sanitizare input, codificare output
  • Protecție CSRF: token-uri anti-CSRF, atribute SameSite
  • Gestionarea JWT: stocare securizată, gestionarea expirării

Securitatea rețelei

  • Toate comunicațiile API peste HTTPS
  • Autentificare și autorizare pentru endpoint-uri protejate
  • Limitarea ratei pentru cererile API

Protecția datelor

  • Stocarea minimă a datelor sensibile pe client
  • Ștergerea datelor sensibile după utilizare
  • Fără stocarea parolelor în text clar

13. Concluzie

Implementarea front-end a aplicației Collect.me oferă o interfață robustă și prietenoasă pentru gestionarea colecțiilor. Aplicația urmează practici moderne de dezvoltare web, cu accent pe design responsive, accesibilitate și securitate. Arhitectura modulară permite întreținere ușoară și extindere viitoare, iar integrarea cu serviciile backend este gestionată printr-un strat API curat, cu gestionare comprehensivă a erorilor și feedback pentru utilizator.

Îmbunătățiri viitoare:

  • Caracteristici de accesibilitate avansate
  • Suport multi-limbă
  • Capacități avansate de căutare și filtrare
  • Caracteristici sociale pentru partajarea colecțiilor
